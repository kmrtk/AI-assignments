# 第1章 インターネットとネットワークの仕組み（12問）

## Q1. インターネットとは

「インターネット」と「Webブラウザで見るホームページ」は同じものですか？違うとすれば、どう違いますか？

**回答**

インターネットとWebブラウザは違うものです。インターネットが世界中のコンピュータ同士をつなげる役目をしていますが、Webブラウザはその情報を閲覧するためのアプリケーションです。

## Q2. IPアドレスとポート番号

このプロジェクトではフロントエンドを `localhost:5173`、バックエンドを `localhost:8080` で起動しています。

- `localhost`とは何を指しますか？
- `:5173` や `:8080` のようなポート番号はなぜ必要ですか？

**回答**

localhostとは自分自身のPCのことを指しており、独自のIPアドレスを保有するものです。通常「127.0.0.1」という表示です。また、データが自身のPC内で完結するため外部の通信網にデータが出ていかないため、他者との干渉が起きません。ポート番号が必要な理由は、IPアドレスのみだとそれがメール用のデータなのか、Web用のデータなのか判断できないため同じIPアドレスからの情報をよりアプリごとに区別するために必要です。

## Q3. DNS

ブラウザのアドレスバーに

`https://www.example.com`

と入力してからページが表示されるまでに、DNSはどのような役割を果たしますか？

**回答**

DNSとは`www.example.com`のようなドメイン名をIPアドレス`192.0.2.1`のように数字に変換する役割を果たすものです。コンピュータ上では私達が使用しているドメイン名では情報を認識できないため、DNSを用いてIPアドレスに変換して情報を認識し処理をしています。

## Q4. HTTPとは

HTTPとは何の略で、どのような役割を持つものですか？

「プロトコル」という言葉の意味も含めて説明してください。

**回答**

HTTPとはHypertext Transfer Protocolの略称です。この役割はWebブラウザとWebサーバーがインターネット上で情報をやり取りするための規定の役割を果たすものです。プロトコルとは異なる機器やソフトウェア間でのデータをやり取りするためのルールの意味合いです。

---

## Q5. HTTPとHTTPSの違い

HTTPとHTTPSの違いは何ですか？

なぜ現在のWebサイトはほぼすべてHTTPSを使用していますか？

**回答**

HTTPとHTTPSはどちらもWebサイトを閲覧する際の通信ルールの役割をはたしていますが、違いは通信が暗号化されているかされていないかの違いです。HTTPでは通信が暗号化されていないため、悪意ある第三者から情報を盗まれる可能性が高いですが、HTTPSでは通信が暗号化されているためセキュリティ上の面でHTTPより安全性が高いです。現在のWebサイトがHTTPSが使用されているのはこのためです。

## Q6. リクエストとレスポンス

Webにおける「リクエスト」と「レスポンス」の関係を、日常生活の例えを使って説明してください。

**回答**

Web通信におけるリクエストとはサーバーにデータの要求をすることであり、レスポンスとはサーバーから要求した情報が処理されてくることです。例：部下に仕事を依頼する⇒部下から仕事の成果報告を受ける。

## Q7. HTTPメソッド

このアプリのAPIでは以下を使っています。

- GET
- POST
- PUT
- DELETE
- PATCH

これらの「HTTPメソッド」とは何ですか？

日常生活の例えで、それぞれの違いを説明してください。

**回答**

HTTPメソッドとはブラウザからサーバーへどんな処理をしたいかをリクエストすることで、代表的な5つのメソッドがあります。GET⇒サーバーからデータを取得する（部下から仕事の成果報告を受ける）。POST⇒サーバーへデータを送信し、新しいリソースを作成する（部下へ新たに仕事を依頼する）。PUT⇒既存データを新しいデータに全て置き換える（部下へ新しい方法で既存の仕事をやり直すよう依頼する）。DELETE⇒サーバー上のデータを削除する（部下へ既存の仕事を取りやめるよう伝える）。PATCH⇒既存データの一部のみ更新する（部下へ既存の仕事を一部やり直すよう伝える）。

## Q8. HTTPステータスコード

サーバーからの返事には「ステータスコード」という3桁の数字が含まれます。

以下のコードはそれぞれ何を意味しますか？

- 200
- 201
- 400
- 404
- 500

**回答**

ステータスコードとはブラウザからリクエストに対しての処理結果を伝える3桁の数字のことです。200⇒リクエストが正常に処理され、データが正しく返された状態。201⇒リクエストが正常に処理され、サーバー上に新しいリソースが作成された状態。400⇒リクエストの仕方に問題がある。404⇒指定したリソースが存在しない。500⇒サーバー側で予期せぬエラーが発生した。

## Q9. JSONとは

フロントエンドとバックエンド間でデータをやり取りする際に「JSON」という形式が使われています。

- JSONとは何ですか？
- なぜ広く使われていますか？

**回答**

JSONとはJavaScript Object Notationの略称で、JavaScriptのプログラミング言語におけるオブジェクトの書き方を参考に作られたデータフォーマットのことです。元々はJavaScriptで使用することを想定して開発されていましたが、人間やコンピューターにとって読みやすく、データが重くなりづらい点や、他の言語に依存しないという点で広く使用されています。

## Q10. APIとは

このアプリではフロントエンドがバックエンドのAPIを呼び出してデータを取得しています。

- APIとは何ですか？
- なぜ直接データベースにアクセスしないのですか？

**回答**

APIとはApplication Programming Interfaceの略称で、ソフトウェアやシステム同士をつなぐ橋渡し役を担っています。私達が料理を注文するときにメニューを参照して店員さんに注文を伝え、店員が注文を厨房の料理人に伝えますが、その流れの中で注文を受け付ける店員の位置づけです。フロントエンドからデータベースに直接アクセスすることは、セキュリティの観点から危険性があるためです。そのため、APIを経由してアクセスします。

---

## Q11. CORSとは

フロントエンド（5173）からバックエンド（8080）にアクセスする際、

「CORS」という仕組みが関係します。

CORSとは何のためにありますか？

**回答**

悪意あるサイトが別のサイトにアクセスするのを防ぐため、異なるオリジンからのアクセス（プロトコル、ドメイン、ポート番号）は制限される可能性があり、その制限を緩和するためのものです。CORSを用いて、各オリジンからのアクセスを許可すると設定することができます。

## Q12. クライアントとサーバー

「クライアント」と「サーバー」とは何ですか？

このアプリの構成の中で、それぞれどれがクライアントでどれがサーバーですか？

- ブラウザ
- バックエンド
- データベース

**回答**

クライアントとはサービスを要求する側であり、サーバーはサービスを提供する側です。ブラウザ⇒クライアント。バックエンド⇒クライアント兼サーバー（ブラウザからはサーバーとしてリクエストを受けますが、DBに対しては処理を要求しているためクライアントとしての役割も担います）。データベース⇒サーバー。

## Q14. 3層構造

このアプリは

- フロントエンド
- バックエンド
- データベース

の3層構造です。

なぜこのように役割を分けるのですか？

1つのプログラムですべてをやらない理由を説明してください。

**回答**

3層構造にするメリットとして、保守性の向上や柔軟な拡張性、セキュリティの強化が挙げられます。それぞれ保守性の向上では各層が独立するため各層それぞれで変更でき他の層に影響を与えずにできます。柔軟な拡張性により、アクセス集中時の負荷状況に合わせ、必要な層のサーバーだけ単独で強化することができ、セキュリティ面ではユーザーが直接DBに触れないため不正アクセスや改ざんなどのリスクを防ぎやすくなるため3層構造にしています。

---

## Q15. データの流れ

ユーザーが「新しいタスクを作成」ボタンを押してから、

- データベースに保存される
- 画面に反映される

までの流れを順を追って説明してください。

**回答**

タスク作成⇒フロントエンドからバックエンドへタスク作成の処理を依頼⇒バックエンドでフロントエンドからの要求を処理⇒バックエンドからDBに作成されたタスクを保存処理し、DBから保存された情報を返してもらうようにリクエスト⇒DBはバックエンドからの保存処理の要求を処理し、バックエンドへ処理結果を返す⇒バックエンドにDBからの処理結果をうけとり、フロントエンドへその処理結果を返す⇒フロントエンドで処理された結果をブラウザ上で表示⇒タスクが作成された

## Q16. エンドポイントとURL設計

- `/api/tasks/{id}`

{id}の部分は各タスク個別を識別するときの番号です

## Q17. 同期処理と非同期処理

「同期処理」と「非同期処理」の違いを、日常生活の例えで説明してください。

なぜWeb通信では非同期処理が必要ですか？

**回答**

同期処理とはタスクを順番に処理していくことですが、非同期処理は複数のタスクを並行して処理し後から結果を取得します。画面操作をフリーズすることなく操作できるUIの維持や、同期処理だと他のリクエストが処理できなくなり効率的な活用や処理性能の向上の観点から非同期処理は必要と考えられているからです。
同期処理⇒アイドルの握手会で順番がくるまで並んで待つ
非同期処理⇒アプリをインストールしている間にWebで他の調べものをする

## Q19. 入力チェック（バリデーション）の多重防御

## Q30. ブランチとPull Request

- ブランチとは何ですか？
- Pull Request の目的は何ですか？
- なぜ全員が main に直接変更しないのですか？

**回答**

ブランチとは、メインの変更履歴から枝分かれさせ、他の作業に影響を与えずに作業するためのスペース的なものです。Pull Requestはブランチで作成した変更をmainに加えるために他の開発者に依頼、確認してもらうための機能です。mainに直接変更を加えてしまうと、今現状動いてるアプリなどが突然動作しなくなるなどの問題が起きます。mainは常に動作しているという事を意識し、ブランチを切って作業することが大切です。

## Q32. なぜ技術的な仕組みを理解すべきか

AI時代においてコードはAIが書けるようになりました。

それでも

- ネットワーク
- データベース
- Webアプリケーションの仕組み

を理解する必要がある理由を、あなたの考えで述べてください。

**回答**

AIにアプリを作ってといえば誰でも作れる時代です。ですが、アプリを開発していくなかで自分の理解度が低いとAIが危険な設計を出しても気づけず(DBとフロントエンドを直接つなぎこんでしまうなど）、大きな問題になりかねないと考えます。AIを使って開発されたアプリの安全性を判断するのは人間の役割であると考えており、その仕組みを理解しておく必要性があると考えます。
